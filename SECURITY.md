# Security Policy

## Public Repository Scope

This repository is a public documentation repository. It intentionally contains no runtime source code, credentials, private prompts, environment configuration, database schema, authentication logic, operational logs, raw user data, or deployment instructions.

## Reporting Security Issues

Do not open public issues containing secrets, private prompts, logs, credentials, user data, or exploit details.

If you need to report a concern, provide a short public-safe summary and request a private disclosure channel from the repository owner.

## What Is Not Included

The public repository excludes:

- API keys, tokens, passwords, and credentials
- authentication and session implementation details
- private model-provider configuration
- storage schemas, table layouts, vector collection details, and local paths
- raw memory records, raw conversation history, job artifacts, and operational logs
- private prompt chains and governance directives
- implementation routes, function names, and cloneable code paths

## Expected Security Boundary

All examples are fictional and sanitized. Public diagrams are conceptual. They do not describe exact deployment topology, network controls, production storage, model routing, or access-control implementation.

## Human Review

Security-sensitive uses of AI output require human review. Synthetic OS documentation should not be interpreted as an endorsement for automated security decisions without qualified review.

# Sanitized OpRep Example

## Purpose

This fictional OpRep-style example shows the public shape of operational reporting without disclosing raw logs.

## Example

```text
Operational Summary

Workflow: Public documentation revision
Status: completed
Context sources: active conversation, retrieved project context
Persistence: final user-facing result eligible for selected logging
Limitations: no private source code or prompt text included in public output
Human review: documentation owner should review before publication
```

## What Is Not Included

This is not a raw OpRep. It excludes timestamps, local paths, model identifiers, raw prompts, raw responses, job payloads, logs, user data, and storage details.

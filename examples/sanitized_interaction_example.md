# Sanitized Interaction Example

## Scenario

A user asks Carter to revise a public architecture summary.

## Public-Safe Flow

1. CRM provides the recent task context.
2. AMS may recall prior public-documentation preferences.
3. RAG may retrieve relevant project knowledge.
4. PGM assembles a governed context.
5. Carter produces a concise revised summary.
6. The final answer may be persisted if the workflow enables persistence.

## Sanitized Output Shape

```text
The revised summary now describes Carter as the flagship Synthetic OS implementation, separates memory from factual verification, and keeps private prompts and storage details out of the public document.
```

## What Is Not Included

No real user prompt, memory record, private output, raw log, route name, source code, or prompt text is included.
